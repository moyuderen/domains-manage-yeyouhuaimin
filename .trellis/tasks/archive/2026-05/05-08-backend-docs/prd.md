# 编写项目后台功能使用文档

## Goal

为域名管理平台编写一份精简的后台功能使用文档，面向平台管理员和运营人员，每个页面用一句话说明功能和特色，配截图展示。

## What I already know

### 项目概况
- 技术栈：Next.js App Router + Shadcn ui + TypeScript + Supabase
- 用途：管理域名、DNS 记录、账户和站点，通过仪表盘监控域名风险
- 认证方式：基于 access key 的登录认证

### 后台功能模块（8 个）

1. **登录** (`/login`) — Access Key 验证入口
2. **看板** (`/dashboard`) — 域名/账户/站点数据图表
3. **域名管理** (`/domains`) — 域名 CRUD、子域名、批量操作
4. **站点管理** (`/sites`) — 第三方站点维护、收藏排序
5. **账户管理** (`/accounts`) — 账号管理、站点关联
6. **设置与通知** (`/settings`) — 项目信息、通知规则、三大通知通道
7. **操作记录** (`/logs`) — 审计追踪、变更对比
8. **任务执行记录** (`/job-runs`) — 定时任务历史、手动触发

## Decisions

| 决策 | 选择 | 原因 |
|------|------|------|
| 文档详细度 | 每页两句（功能+特色） | 用户偏好精简，不需要详细操作步骤 |
| 截图方式 | Puppeteer 无头浏览器自动截取 | 高效，16:9 统一比例 |
| 文档落点 | 合并到 README.md | 用户要求，方便开源展示 |
| Mock 数据 | 新建 `lib/mock/` + data 层回退 | 操作日志和任务执行无真实数据，需要展示效果 |
