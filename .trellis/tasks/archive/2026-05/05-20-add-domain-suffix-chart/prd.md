# 域名后缀筛选与后缀分布图表

## Goal

围绕“域名后缀”增加两类能力：

1. 在域名列表页支持按后缀筛选，方便快速定位某类后缀域名。
2. 在域名监控看板新增“后缀分布”柱状图，直观看到各后缀的数量，并在悬浮时预览该后缀下的前 10 个域名。

整体实现保持与现有项目一致：列表页继续采用 URL 参数驱动 + 服务端过滤；看板图表复用现有 Recharts + shadcn 图表模式。

## MVP Requirements

### 1. 域名列表页后缀筛选

* 仅支持单个后缀筛选，不支持多后缀并集。
* UI 同时支持：
  * 手动输入后缀
  * 从下拉中选择后缀
* 手动输入和下拉选择共用同一个筛选字段与 URL 参数语义。
* 下拉候选后缀从全量域名数据中动态汇总，不使用固定枚举列表，也不跟随当前筛选结果变化。
* 后缀筛选可与现有关键字、状态、注册站点、DNS 站点、排序、分页一起使用。

### 2. 域名监控看板后缀分布图表

* 在域名监控看板新增一个“域名后缀分布”柱状图。
* 图表展示所有后缀及对应数量，不只展示 Top N。
* 每个柱条显示该后缀的域名数量。
* 悬浮到柱条时，tooltip 展示：
  * 后缀名称
  * 该后缀的域名总数
  * 该后缀下前 10 个域名
* tooltip 交互与视觉风格参考现有“注册商分布”图表。
* 点击柱条后，跳转到 `/domains?suffix=...` 的筛选结果页，方便直接查看该后缀的域名列表。
* 图表数据与列表页后缀筛选使用同一套后缀定义，避免口径不一致。

## Matching Rule

后缀能力统一采用业务规则“字符串尾部匹配”，不引入公共后缀解析库。

* 输入 `com` 时，匹配所有以 `.com` 结尾的记录。
* 输入 `mc.cc` 时，匹配所有以 `.mc.cc` 结尾的记录。
* 域名记录可能包含子域名前缀，例如 `www.a.com`、`api.foo.mc.cc`，这些记录也应按尾部规则正确匹配。
* 图表中的后缀分组也按同一规则生成，例如：
  * `www.a.com` 归入 `com`
  * `api.foo.mc.cc` 归入 `mc.cc`

## Acceptance Criteria

### 列表页

* [ ] 用户可以在域名列表页通过输入或下拉选择一个后缀进行筛选。
* [ ] 输入 `com` 时，可匹配 `a.com`、`www.a.com` 等以 `.com` 结尾的记录。
* [ ] 输入 `mc.cc` 时，可匹配 `foo.mc.cc`、`api.foo.mc.cc` 等以 `.mc.cc` 结尾的记录。
* [ ] 后缀筛选可与现有其他筛选条件叠加使用。
* [ ] 当前后缀筛选状态可通过 URL 参数表达，并在刷新后恢复。
* [ ] 下拉候选来自全量域名汇总，且不会因当前筛选结果变化而波动。

### 看板

* [ ] 域名监控看板新增“域名后缀分布”柱状图。
* [ ] 图表展示所有已出现的后缀及其数量。
* [ ] 悬浮任一柱条时，tooltip 可展示该后缀下前 10 个域名。
* [ ] 点击柱条后，可跳转到对应后缀的域名列表筛选结果页。
* [ ] 图表中的数量与列表页按相同后缀筛选得到的结果数量一致。

## Out of Scope

* 多后缀同时筛选
* 基于 Public Suffix List 的标准后缀解析
* 域名新增/编辑表单、详情页改造
* 后缀图表以外的新看板筛选器或复杂联动

## Technical Approach

* 在 `types/domain.ts` 中为 `DomainFilters` / `DomainListQuery` 增加 `suffix` 字段。
* 在 `app/(main)/domains/page.tsx` 中读取并传递 `suffix` URL 参数。
* 在 `components/domains/DomainsPageClient.tsx` 中统一维护 `suffix` 与其他筛选项的 URL 同步。
* 在 `components/domains/DomainToolbar.tsx` 中新增一个可输入、可选择的后缀筛选控件。
* 在 `lib/data/domains.ts` 中为 Supabase 查询和 mock 数据过滤补充统一的尾部匹配逻辑。
* 在 `lib/dashboard.ts` 中新增“按后缀聚合”的统计函数，输出图表需要的后缀名、数量和前 10 个域名。
* 在 `lib/data/dashboard.ts` 中把后缀分布数据加入 dashboard 数据模型。
* 在 `components/dashboard/OverviewCharts.tsx` 和现有图表组件中新增后缀分布图表，尽量复用“注册商分布”的柱状图、tooltip 与点击跳转模式。

## Technical Notes

* 现有“注册商分布”实现位于 `components/dashboard/ProviderChart.tsx:63`，tooltip 已展示站点名称、数量和部分域名列表，可直接复用交互模式。
* 当前 `OverviewCharts` 位于 `components/dashboard/OverviewCharts.tsx:15`，采用两列网格布局，新增图表后需要确认布局是否仍然自然。
* 当前 dashboard 数据入口位于 `lib/data/dashboard.ts:9`，域名统计聚合位于 `lib/dashboard.ts:25` 之后。

## Implementation Plan

### 子需求 1：域名列表页后缀筛选

* 为列表页增加 `suffix` 筛选参数与 UI。
* 支持手动输入 + 下拉选择。
* 完成 URL 同步、服务端过滤、全量候选后缀汇总。
* 先把“后缀口径”和筛选行为跑通。

### 子需求 2：域名监控看板后缀分布图表

* 在看板新增“域名后缀分布”柱状图。
* 复用子需求 1 中已经确定的后缀定义与分组逻辑。
* 展示每个后缀的数量，并在 tooltip 中展示该后缀下前 10 个域名。
* 与列表页保持统计口径一致，并支持点击柱条跳转到对应的列表筛选结果。

## Decision

本次采用轻量业务规则实现域名后缀能力：不依赖后缀库，不做标准公共后缀解析，统一按业务定义的后缀字符串进行尾部匹配与分组。列表页和看板共用同一套后缀口径，保证筛选结果与图表统计一致。
